Le projet

Une seule architecture, quatre expériences

Plutôt que de proposer des bornes séparées (lobby, satisfaction, room service), Concierge fédère ces touchpoints autour d'un backend NestJS unifié, d'une base MongoDB Atlas multi-tenant, et d'un déploiement live sur Render et Vercel.

Architecture technique

Une stack monorepo, cinq interfaces, six services

Tout passe par un API Gateway unique. Les microservices NestJS partagent une base MongoDB Atlas avec isolation par tenantId. Temps réel via Socket.io pour la borne lobby et la réception.

01

Multi-tenant par discriminator

Une seule base, un seul déploiement. Chaque collection porte un tenantId. Onboarder un nouvel hôtel = 0 ops.

02

Frontends spécialisés

Vue + Ionic pour les bornes touch-first, Angular Signals pour les dashboards form-heavy. Tokens design partagés via @concierge/design-tokens.

03

Temps réel via Socket.io

Quand une commande est créée sur la borne, elle pousse instantanément vers la réception. Room tenant-{id} pour l'isolation.

04

Sécurité par défaut

JWT 15 min + refresh rotation, Helmet + HSTS preload, Throttler 100 req/min sur auth, validation pipe, RGPD anonymisation, audit log.

05

Observabilité

/healthz + /readyz sur chaque service, logs JSON structurés au gateway avec correlation-id et tenant-id, Swagger 3.0 agrégé.

06

Déploiement minimal

Vercel pour les 5 frontends, Render free tier pour le backend, MongoDB Atlas M0. buildFilter par service pour des deploys ciblés. Coût total : 0 €.
